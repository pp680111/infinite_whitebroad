## 1. 概述

### 1.1 产品定位

个人知识卡片收藏夹 — 在无限画布上以自由布局的方式管理碎片化的想法、笔记和资料。

### 1.2 核心体验

- 打开应用即进入无边界的画布
- 快速创建卡片（快捷键或点击）→ 键入内容 → 继续浏览
- 卡片像便签一样散落在画布上，靠拖拽重新组织
- 不需要目录结构，靠位置和视觉搜索找到内容
- 支持卡片内文本内容的全文搜索

### 1.3 设计语言

- **纸质感**：卡片有微妙的阴影、浅米色/白色背景
- **字体**：无手写字体，使用无衬线字体（Inter / Noto Sans）
- **风格**：简洁，工具栏最小化，主要靠快捷键操作
- **画布背景**：浅灰格点，辅助定位

### 1.4 目标平台

- 桌面端（Electron）

### 2.3 IPC 通信设计

| 通道 | 方向 | 用途 |
|------|------|------|
| `file:new` | renderer → main | 新建空白画布 |
| `file:open` | renderer → main | 打开文件对话框，选择后加载文件内容 |
| `file:save` | renderer → main | 保存到当前文件 |
| `file:save-as` | renderer → main | 另存为新文件 |
| `file:get-recent` | renderer → main | 获取最近文件列表 |
| `file:add-recent` | renderer → main | 添加文件到最近列表 |

> **说明**：所有通道均为请求-响应模式。renderer 主动发起请求，main 处理后返回结果；main 不会主动推送数据。

### 2.4 主进程与渲染进程职责

- **Electron 主进程**：窗口管理、原生菜单、文件系统读写、快捷键注册
- **Renderer 进程**：Fabric.js 画布渲染、React UI 组件、状态管理

## 3. 画布系统

### 3.1 无限画布模型

- 使用 Fabric.js 的 viewport transform 实现缩放/平移
- 默认缩放范围：10% ~ 400%
- 画布坐标系统：所有元素使用世界坐标系，viewport 变换仅影响视觉
- 双击空白区域 → 创建新卡片
- 滚轮缩放，**以鼠标指针位置为中心**；空格+拖拽平移
- **性能策略**：Fabric.js 默认渲染所有元素；当元素超过 200 个时，应实现视口裁剪（使用 Fabric.js `clipPath` 或自定义渲染逻辑，仅渲染可见区域内的元素）
- **viewport 状态**：属于文档元数据的一部分，随文件保存和加载（见 5.1 节）

### 3.2 元素类型

| 类型 | 说明 |
|------|------|
| 卡片 (Card) | 主容器，内含文本/图片，背景米白；工具栏「文本卡片」即创建此元素 |
| 矩形 (Rectangle) | 装饰性/区域划分用，描边无填充 |
| 椭圆 (Ellipse) | 同上，圆形/椭圆形装饰 |
| 图片 (Image) | 独立图片元素（PNG/JPG/GIF/WebP），作为画布一级对象；与卡片内图片是不同概念 |

> **工具栏说明**：工具栏「文本卡片」按钮 = 创建 Card 元素；「图片」按钮 = 创建独立 Image 画布元素。两者是不同的元素类型。

### 3.3 元素交互

- 单击选中，拖拽移动
- 选中后四角/边线控制缩放
- 旋转控制点（可选）
- 右键菜单：删除、复制、置顶/置底、锁定
- **锁定行为**：锁定后元素不可移动、不可缩放、不可编辑（双击无效）、不可删除，但仍可通过右键菜单解锁；锁定元素仍可被选中（显示选中框但不显示控制手柄）；尝试拖拽锁定元素时：选中框轻微晃动（视觉反馈）但位置不变
- 多选：Shift+点击，或框选（仅在选择工具下，拖拽空白区域创建选区）；若当前非选择工具，拖拽空白区域执行平移

## 4. 卡片组件

### 4.1 卡片结构

```
┌─────────────────────────┐
│ [标题]              [⋮] │  ← 可选标题栏，⋮是操作菜单
│ ─────────────────────── │
│                         │
│  内容区域                │  ← 文本/图片混排
│  支持多行文本            │
│                         │
│  [图片区域]              │
│                         │
└─────────────────────────┘
```

### 4.2 卡片属性

| 属性 | 说明 |
|------|------|
| id | UUID |
| position | { x, y } 世界坐标 |
| size | { width, height } |
| title | 可选标题 |
| content | HTML 富文本，经过 sanitization 白名单过滤：允许 <p>, <br>, <strong>, <em>, <u>, <s>, <h1-h6>, <ul>, <ol>, <li>, <blockquote>, <code>, <pre>, <a>，其余标签和属性全部移除，<script>、<style>、<iframe> 等危险标签及事件属性永久移除 |
| images | 图片引用列表 [{ id, src, position }]，src 为 Base64 字符串（data:image/...格式） |
| createdAt | 创建时间 |
| updatedAt | 更新时间 |

### 4.3 卡片交互

- 双击卡片 → 进入编辑模式
- 编辑模式下可添加/编辑文本、插入图片
- 文本超出卡片边界时：**卡片高度自动撑高**，卡片最小宽度固定（200px），最大宽度不超过原始宽度（超出部分换行）
- 多张图片在卡片内：**垂直堆叠排列**，间距 8px
- 点击已插入图片右上角删除按钮 → 移除图片
- 点击空白处 → 退出编辑，保存
- 拖拽图片到卡片内 → 追加到卡片
- `Ctrl+V` 粘贴 → 粘贴剪贴板内容（文本或图片）到卡片内

## 5. 持久化与文件管理

### 5.1 文件格式

- **扩展名**：`.whiteboard`
- **格式**：JSON

```json
{
  "version": "1.0",
  "metadata": {
    "name": "我的知识库",
    "createdAt": "2026-03-20T10:00:00Z",
    "updatedAt": "2026-03-20T10:00:00Z",
    "viewport": { "x": 0, "y": 0, "zoom": 1 }
  },
  "canvasState": { /* Fabric.js JSON */ },
  "searchIndex": [ /* 纯文本索引用于搜索 */ ]
}
```

- 加载文件时自动恢复 viewport（画布位置和缩放级别）

### 5.2 文件操作

| 操作 | 触发 | 说明 |
|------|------|------|
| 新建 | `Ctrl+N` | 创建空白画布（内存中），不立即保存；标题显示"未命名" |
| 打开 | `Ctrl+O` | 文件对话框选择 `.whiteboard`，加载后替换当前画布 |
| 保存 | `Ctrl+S` | 若为未命名文档，弹出保存对话框；否则保存到当前文件 |
| 另存为 | `Ctrl+Shift+S` | 弹出保存对话框，保存为新文件 |
| 最近文件 | 启动页显示 | 最多显示 10 个；文件已删除或路径无效时显示但禁用；文件被移动/重命名后视为无效 |

### 5.3 自动保存

- 自动保存仅对**已保存过的文档**生效（未命名文档不触发 auto-save）
- 编辑触发后开始计时，**30 秒内无任何画布相关操作**则自动保存
- 「操作」定义：鼠标点击/拖拽、画布缩放/平移；键盘输入会重置计时器
- **卡片编辑模式下完全暂停计时**：用户正在编辑卡片时计时器停止，退出编辑模式后重新开始计时；即使用户编辑 10 分钟，自动保存也不会在编辑期间触发，只在退出编辑后等待 30 秒无操作才触发
- 状态栏显示"已保存"/"保存中..."
- 崩溃恢复：保存时在同目录下创建 `.tmp` 临时文件（自定义实现，非 electron-store 内置机制）；下次启动时检测该文件是否存在，如存在则提示用户恢复；恢复成功后删除 `.tmp` 文件

### 5.4 关闭应用时的未保存内容处理

- 关闭应用时若存在未保存的更改，显示确认对话框：
  - "保存" → 保存后退出
  - "不保存" → 放弃更改，退出
  - "取消" → 取消关闭操作
- 若文档从未保存过（未命名），关闭时等同于"另存为"流程

### 5.5 错误处理策略

| 场景 | 处理方式 |
|------|----------|
| 手动保存失败（磁盘满/权限问题） | 弹出错误提示，保留文档在内存中，用户可尝试另存为 |
| 自动保存失败 | 静默重试 3 次，间隔 5 秒；3 次仍失败则忽略，不打扰用户；下次手动保存时提醒 |
| 打开无效文件 | 弹出错误提示，不加载内容 |
| 文件损坏（JSON 解析失败） | 弹出错误提示，不加载内容 |
| viewport 数据非法（NaN、非数字、超出范围） | 使用默认值 { x: 0, y: 0, zoom: 1 } |
| canvasState 损坏（Fabric.js 结构不匹配） | 弹出错误提示"画布内容损坏，无法加载"，显示空白画布；用户可选择重新开始或尝试修复 |
| 拖入无效图片格式 | 显示 toast 提示"不支持的图片格式" |
| 加载过大 Base64 图片 | 限制单张图片不超过 2MB，超出则提示压缩 |

### 5.6 图片存储

- 所有图片在加载时转为 Base64（data:image/...格式），内嵌在 JSON 中
- 单张图片大小限制：不超过 2MB（超出提示用户压缩后再试）
- 单张图片尺寸限制：宽或高不超过 1920px（超出自动等比缩放）
- PNG/JPEG 图片质量压缩至 80%；GIF/WebP 保持原样（不压缩动画）
- 整个文档图片总数量建议不超过 100 张

### 6.2 顶栏（标题栏区域）

- 使用 Electron **原生窗口标题栏**（非自定义标题栏），标准窗口控件（最小化/最大化/关闭）
- **左**：菜单按钮（`≡`）→ 新建/打开/保存/导出
- **中**：当前文档名称（可点击重命名）
- **右**：搜索按钮、设置按钮

### 6.3 浮动工具栏

- 圆角矩形，底部居中（工具栏中点对准屏幕水平中央）
- 基础工具：选择 / 矩形 / 椭圆 / 图片 / 文本卡片
  - **选择**：默认工具，选中元素
  - **矩形**：在画布上创建独立矩形图形
  - **椭圆**：在画布上创建独立椭圆图形
  - **图片**：在画布上创建独立图片元素（相当于把图片作为画布的一级元素）
  - **文本卡片**：在画布上创建文本卡片（带便签样式的卡片容器）
- 背景：半透明白 + 毛玻璃效果
- 显示在 Canvas 上方，不受画布缩放/平移影响

### 6.4 浮动缩放控制

- 圆角矩形，位于底部右侧，与工具栏同一水平线
- 显示当前缩放比例 + 滑块或按钮调节
- 显示在 Canvas 上方，不受画布缩放/平移影响

## 7. 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Space` + 拖拽 | 平移画布 |
| `Ctrl` + 滚轮 | 缩放 |
| `Ctrl + N` | 新建 |
| `Ctrl + O` | 打开 |
| `Ctrl + S` | 保存 |
| `Ctrl + F` | 搜索 |
| `Delete` | 删除选中 |
| `Ctrl + D` | 复制选中 |
| `Ctrl + A` | 全选 |
| `Escape` | 取消选择/退出编辑 |
| `Ctrl + Z` | 撤销 |
| `Ctrl + Shift + Z` 或 `Ctrl + Y` | 重做 |
| `Ctrl + V` | 粘贴（编辑模式下粘贴文本/图片到卡片） |

### 7.1 撤销/重做机制

- 使用命令模式（Command Pattern）记录操作
- 支持的操作：创建元素、删除元素、移动、缩放、旋转、编辑内容
- **富文本编辑的撤销粒度**：停止输入 1 秒后或点击空白处退出编辑时，将该次编辑内容作为一个撤销步骤（避免每次按键都生成一条记录）
- 历史栈容量：最多 50 步；达到上限后新的撤销操作继续工作（丢弃最早记录）

## 8. 文本搜索

- 保存时额外提取纯文本索引（去除 HTML 标签）
- 搜索：**不区分大小写**，支持**子字符串匹配**
- `Ctrl+F` 唤起搜索面板，输入关键词
- 若当前有元素处于编辑模式，**自动退出编辑**后再定位
- 定位到匹配卡片时：**退出编辑模式**，卡片获得焦点并滚动到可见区域
- 回车定位到下一个匹配，再次回车定位再下一个
- 无匹配结果时显示"未找到"
- 搜索结果高亮显示（仅当前会话有效）
- 关闭搜索面板或按 `Escape` 时清除所有高亮

## 9. MVP 范围

- 无限画布（缩放/平移）
